0 1 — DESIGN SYSTEM
Design
Foundations
Everyone on the team should pull from the same values. Before building any page, set these up
in your tailwind.config.js or as CSS variables. This is how the site stays visually consistent even
though different people are building different pages.
Using AI tools? Always start your prompt with: "I'm building a tech club website. Indigo accent only,
white background, Tailwind CSS, no purple gradients, clean editorial style." Otherwise you'll get
generic output.

Typography
Two fonts only — add them via Google Fonts.
Display / Headings / Body: DM Sans or Syne
Dates, Tags, Code snippets: JetBrains Mono
Size scale matters more than font choice. Establish a clear H1 → body → caption ladder and stick
to it across every page.
// tailwind.config.js

heading: '#18181B', // zinc-900

Spacing
If something looks cramped, add more padding. Whitespace is design — it's not wasted space.
Sections should feel like they have room to exist.
Interactions
Hover: Border-color shift or scale(1.02) — pick one and apply it site-wide.
Micro-interactions: transition: 150ms ease
Layout shifts / page loads: transition: 300ms ease-out
Entry animations run once, fast. Never delay interaction.
Glassmorphism — use it once
Fine for one featured card or hero element. Not for every surface.

0 2 — ARCHITECTURE
Data Pattern — Everyone Must
Follow
Instead of hardcoding text inside your components, put your data in a separate JS/JSON file and
loop through it. This means: to update content, you edit one file. Your component doesn't
change. It also means your teammate can easily add new events, members, or blogs without
touching any code logic.
Folder structure
Example — events.js
src/
├── data/
│ ├── events.js ← Sameer edits this
│ ├── members.js ← Advik edits this

How to use it in a page
Same pattern for: members, blogs, projects. Data lives in /src/data/ . Components in /
components/ . Pages just import and render.
// app/events/page.tsx
import { events } from '@/data/events'
export default function EventsPage() {
const upcoming = events.filter(e => e.tag === 'upcoming')
const past = events.filter(e => e.tag === 'past')
return (
 <>
 <section>
 <h2>Upcoming</h2>
 {upcoming.map(event => (
 <EventCard key={event.id} {...event} />
 ))}
 </section>
 <section>
 <h2>Past</h2>
 {past.map(event => (
 <EventCard key={event.id} {...event} />
 ))}
 </section>
 </>
 )
}

0 3 — PER-PAGE SPECS
Per-Page
Guide
Ashmit — Landing Page, Loader,
Footer
app/page.tsx · Hero.tsx · Loader.tsx ·
Footer.tsx
Loader
Keep it under 1.5 seconds. A pulsing indigo bracket { } or a thin progress bar across the
top is enough. Don't build a cinematic intro — it annoys users on repeat visits.
Hero
Large display text, a strong subline, two buttons, and a floating preview card for depth.
Layout:
[Big Headline — 2 lines max, text-6xl+]
[One-line subline describing the club]
[Button: Join Us]  [Button: Explore →]
         [Floating stats card or event preview]
One filled button (indigo bg) + one ghost button (indigo outline)
Below the fold: render <EventsPreview /> , <MembersPreview /> , <BlogsPreview /> — 3 items
max each, with a "View All →" link
Footer
Three columns: Logo + tagline / Quick links / Social icons. border-t border-zinc-200 as
separator. No gradients.

0 4 — RULES
What to
Avoid
Don't do this Why
gradient backgrounds (purple/violet) Looks like every AI-generated site ever
Inter or Roboto as the only font Generic — use what's specified above
Glassmorphism on every card Use it once, on one hero element
More than one accent color Kills visual cohesion across pages
Animations that fire on every scroll Gets annoying fast, hurts performance
Different border-radius values per page Inconsistent — pick rounded-xl everywhere
Hardcoded text in JSX components Edit one data file, not five component files
Images without fixed aspect ratio Cards break on different screen sizes
Final
Checklist
Before calling a page done, ask yourself:
Does all content come from the /src/data/ file, not hardcoded in JSX?
Is every repeated UI element (card, badge, button) a shared component?
Are you only using #4F46E5 as the accent color?
Does the page look consistent with the rest of the site (same spacing, font scale, borderradius)?
Would a stranger understand what this club does within 5 seconds on the landing page?
Would a CS student want to screenshot this for their portfolio?
That last question is the bar. If yes, ship it.
